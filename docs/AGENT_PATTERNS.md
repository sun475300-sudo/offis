# AI 에이전트 협업 패턴 가이드

> **11 에이전트 기반 협업 아키텍처** — 단독 실행부터 전체 오케스트레이션까지

---

## 🏗️ 11 에이전트 전체 구조

```
                    ┌─────────────┐
                    │  비서 (Secretary) │  ← 오케스트레이터
                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ 리서처   │    │  분석가  │    │  작성자  │
    └──────────┘    └──────────┘    └──────────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │  운영자  │    │ 디자이너 │    │  코더   │
    └──────────┘    └──────────┘    └──────────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │  테스터  │    │ 리뷰어   │    │옵티마이저│
    └──────────┘    └──────────┘    └──────────┘
                    ┌──────────┐
                    │ 번역가   │  ← 독립 지원
                    └──────────┘
```

## 📐 핵심 협업 패턴 11가지

### 패턴 1: 순차 파이프라인 (Sequential Pipeline)
```
트리거 → A → B → C → 결과물
```
**사용 사례**: 영상 에세서, 제안서 생성
```
운영자(데이터 수집) → 분석가(인사이트) → 작성자(콘텐츠) → 운영자(저장)
```
**특징**: 각 단계 산출물이 다음 단계 입력. 오류 시 해당 단계부터 재시작.

---

### 패턴 2: 병렬 팬아웃 (Parallel Fan-out)
```
트리거 → 비서 → [A, B, C 동시] → 비서(통합) → 결과물
```
**사용 사례**: 경쟁사 3사 동시 조사
```
비서 → ┌─ 리서처(경쟁사 A)
       ├─ 리서처(경쟁사 B)  → 분석가(통합 비교) → 작성자(보고서)
       └─ 리서처(경쟁사 C)
```
**특징**: 독립 작업 병렬화로 시간 단축. 비서가 결과 수집·통합.

---

### 패턴 3: 검증 루프 (Validation Loop)
```
코더(구현) → 테스터(검증) → [통과 → 릴리즈 | 실패 → 코더(수정)]
```
**사용 사례**: 기능 개발 + 테스트 사이클
```
코더 → 테스터 → 리뷰어
  ↑         │
  └─ 수정 ──┘ (실패 시 반복)
```
**특징**: 품질 게이트 보장. 최대 3회 루프 후 에스컬레이션.

---

### 패턴 4: 전문가 체인 (Expert Chain)
```
분석가(문제 정의) → 옵티마이저(해결책) → 코더(구현) → 테스터(검증)
```
**사용 사례**: 성능 이슈 해결
```
옵티마이저(병목 분석) → 코더(최적화 구현) → 테스터(성능 측정)
측정 결과 → 옵티마이저(추가 개선 여부 판단)
```

---

### 패턴 5: 이중 검토 (Dual Review)
```
코더(구현) → 테스터(기능 검증) + 리뷰어(코드 품질) → 머지
```
**사용 사례**: 중요 기능 PR
```
코더 → ┌─ 테스터(모든 테스트 통과?)
       └─ 리뷰어(Critical 이슈 없음?)
            ↓ 둘 다 통과
         운영자(커밋 + PR 생성)
```

---

### 패턴 6: 콘텐츠 증폭 (Content Amplification)
```
1개 원본 → 분석가 → 작성자 → N개 파생 콘텐츠
```
**사용 사례**: SNS 멀티플랫폼 배포
```
영상 1개
→ 분석가(핵심 메시지 추출)
→ 작성자 ┌─ YouTube Shorts 스크립트
          ├─ Instagram 캡션
          ├─ LinkedIn 게시물
          ├─ Twitter/X 쓰레드
          ├─ 블로그 포스트 1500자
          └─ 뉴스레터 섹션
```

---

### 패턴 7: 번역 브릿지 (Translation Bridge)
```
한국어 콘텐츠 → 번역가 → 영어 버전 → 글로벌 배포
```
**사용 사례**: 글로벌 팀 협업, 오픈소스 기여
```
작성자(한국어 문서) → 번역가 → 작성자(영어 검수) → 운영자(배포)
```
**적용 범위**: README, API 문서, 에이전트 프롬프트, UI 텍스트

---

### 패턴 8: 릴리즈 오케스트레이션 (Release Orchestration)
```
비서 → [코더·테스터·리뷰어 동시] → 옵티마이저 → 운영자(배포)
```
**사용 사례**: 버전 릴리즈 전체 자동화
```
비서 → ┌─ 코더(tsc --noEmit 검증)
       ├─ 테스터(npm test 전체 통과)
       └─ 리뷰어(코드 리뷰 최종 확인)
            ↓
       옵티마이저(성능 회귀 없음 확인)
            ↓
       운영자(changelog → release notes → git tag → push)
```

---

### 패턴 9: 인텔리전스 루프 (Intelligence Loop)
```
리서처(정보 수집) → 분석가(패턴 발견) → 비서(전략 수정) → 반복
```
**사용 사례**: 지속적 경쟁사 모니터링
```
매주 월요일:
리서처(경쟁사 동향) → 분석가(변화 감지) → 작성자(브리핑) → 비서(전략 업데이트 여부 판단)
```

---

### 패턴 10: 점진적 정교화 (Iterative Refinement)
```
초안 → 리뷰어(피드백) → 수정 → 리뷰어(재검토) → 완성
```
**사용 사례**: 고품질 문서, 제안서 작성
```
작성자(초안) → 리뷰어(개선점 N개)
    ↓
작성자(수정 v2) → 리뷰어(Critical 해결 확인)
    ↓
번역가(영어 버전) → 완성
```

---

### 패턴 11: 비상 대응 (Emergency Response)
```
이슈 감지 → 비서(즉시 할당) → 코더+테스터(병렬) → 운영자(핫픽스)
```
**사용 사례**: 프로덕션 버그, 빌드 실패
```
빌드 실패 알림
    ↓
비서(심각도 판단: P1/P2/P3)
    ↓ P1
코더(원인 분석) + 테스터(재현 확인) [동시]
    ↓
코더(핫픽스) → 테스터(검증) → 운영자(긴급 배포)
    ↓
리뷰어(사후 원인 분석 리포트)
```

## 🎯 패턴 선택 가이드

| 상황 | 권장 패턴 |
|------|-----------|
| 콘텐츠 제작 | 순차 파이프라인 + 콘텐츠 증폭 |
| 복수 리서치 | 병렬 팬아웃 |
| 기능 개발 | 검증 루프 + 이중 검토 |
| 성능 이슈 | 전문가 체인 |
| 버전 릴리즈 | 릴리즈 오케스트레이션 |
| 글로벌 배포 | 번역 브릿지 |
| 지속 모니터링 | 인텔리전스 루프 |
| 고품질 문서 | 점진적 정교화 |
| 버그/장애 | 비상 대응 |
