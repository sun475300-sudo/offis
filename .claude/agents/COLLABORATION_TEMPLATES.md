# 에이전트 협업 템플릿

> Secretary가 나머지 4개 에이전트를 조율하는 표준 패턴

---

## 템플릿 A: 콘텐츠 마케팅 파이프라인

```
트리거: 사용자 → "[주제] 마케팅 콘텐츠 패키지 만들어줘"

STEP 1  [비서] 태스크 분해 & 위임
        ├─ 리서처에게: "{주제} 시장 조사 + 경쟁사 5개 분석 (출력: research-{주제}.md)"
        ├─ 분석가에게: "리서처 결과 받으면 포지셔닝 전략 도출 (출력: strategy-{주제}.md)"
        └─ 작성자에게: "전략 완료 후 SNS 스킬 실행 (출력: content-{주제}/ 폴더)"

STEP 2  [리서처] → research-{주제}.md 작성 완료 → 비서에게 완료 보고

STEP 3  [분석가] research 파일 수신 → strategy-{주제}.md 작성 완료 → 비서에게 완료 보고

STEP 4  [작성자] strategy 파일 수신 → /skill sns 실행 → 콘텐츠 패키지 생성

STEP 5  [비서] 전체 결과 통합 → 사용자에게 파일 목록 + 요약 보고

예상 소요: 3-8분 (LLM 연동 여부에 따라)
출력물: research-*.md / strategy-*.md / content-*/
```

---

## 템플릿 B: GitHub PR 기술 검토 파이프라인

```
트리거: /github {owner}/{repo} {PR번호}

STEP 1  [비서] GitHub 워크플로 개시 선언

STEP 2  [리서처] + [분석가] 병렬 실행
        ├─ [리서처] PR 변경 파일 목록 + 관련 이슈 조사
        └─ [분석가] 코드 영향 범위 + 리스크 분류

STEP 3  [운영자] 자동화 검증
        ├─ npx tsc --noEmit
        ├─ 린트 실행
        └─ 결과 정리

STEP 4  [작성자] 리뷰 코멘트 초안 작성
        (리서처 + 분석가 + 운영자 결과 종합)

STEP 5  [비서] 최종 리뷰 패키지 → 사용자에게 전달
        + DebateManager 기술 토론 세션 트리거

출력물: pr-review-{PR번호}.md
```

## 템플릿 C: 신규 스킬 자동 패키징

```
트리거: 작업 완료 후 "이 작업 스킬로 저장해줘"

STEP 1  [비서] 직전 완료 작업 단계 역추적 요청

STEP 2  [분석가] 작업 패턴 분석
        ├─ 입력 파라미터 식별
        ├─ 처리 단계 정리
        └─ 출력 형식 정의

STEP 3  [운영자] 스킬 정의 파일 생성
        ├─ .claude/agents/skills/{스킬명}.md 작성
        ├─ CLI 명령어 등록
        └─ 테스트 케이스 추가

STEP 4  [비서] 스킬 등록 확인 + 사용자에게 호출 방법 안내

출력물: .claude/agents/skills/{스킬명}.md
```

## 템플릿 D: 주간 성과 보고서

```
트리거: 매주 금요일 또는 "/report weekly"

병렬 실행:
├─ [분석가] TaskService 완료/실패/진행 중 태스크 집계
├─ [리서처] GitHub 이번 주 머지된 PR + 이슈 목록 조회
└─ [운영자] 빌드 성공률 + 에러 로그 요약

[작성자] 위 3개 데이터 수신 → 보고서 작성
[비서] 최종 검토 + 사용자에게 전달

출력물: weekly-report-{YYYY-WW}.md
```

## 에이전트 간 메시지 포맷

에이전트 간 작업 위임 시 표준 포맷:

```markdown
**FROM**: {발신 에이전트}
**TO**: {수신 에이전트}
**TASK**: {구체적 작업 내용}
**INPUT**: {입력 파일/데이터 경로}
**OUTPUT**: {기대 출력 파일명}
**DEADLINE**: {우선순위: urgent|normal|low}
**DEPENDS_ON**: {선행 완료 필요 태스크 ID (없으면 none)}
```
